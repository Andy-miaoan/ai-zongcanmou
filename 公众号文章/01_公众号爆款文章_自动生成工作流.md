## 二、文章结构模板（AIDA-R，强制使用）

```
┌─────────────────────────────────────────┐
│ A → 开头钩子（3秒代入）                   │
│     具体人物+具体细节+痛点台词             │
│     禁止：观点句开头、恐惧开头             │
├─────────────────────────────────────────┤
│ I → 痛点放大（让读者"认账"）              │
│     数据/案例/场景还原                    │
│     到这里读者应该觉得"说的就是我"          │
├─────────────────────────────────────────┤
│ D → 认知翻转（反常识结论）                │
│     "你以为X，真正的原因是Y"              │
│     紧接着【反思停留】150-200字            │
│     让读者停在这个冲击里，不要急着给方法    │
├─────────────────────────────────────────┤
│ A → 方法拆解（2-4个模块）                 │
│     每个模块：问题→操作→示例              │
│     70-80%的篇幅给方法论本身               │
│     穿插个人踩坑经历（增加真实感）          │
├─────────────────────────────────────────┤
│ R → Retention & Action                  │
│     划重点（4条金句）→ 下篇预告 →          │
│     互动引导 → 关注引导 → 微信CTA          │
└─────────────────────────────────────────┘
```

### 严格比例控制

| 部分 | 占比 | 说明 |
|------|------|------|
| 开头故事+痛点放大 | 20-25% | 让人读下去 |
| 认知翻转+反思停留 | 15-20% | "认账"时刻 |
| 方法拆解 | 50-55% | 核心价值，纯干货 |
| 产品提及 | ≤8% | 只在方法拆解结束后出现一次 |
| 划重点+CTA+互动 | 5-8% | 收尾 |

## 三、开头钩子 · 四种开篇模式（轮换使用）

### 模式A：第一人称踩坑（优先使用，每周至少2篇）

```
"[具体时间]，一个[具体身份的老板]找到我。[他遇到的困境]。

他花了[具体数字]做了[具体动作]，结果[出乎意料的结果]。

他跟我说了一句话，让我记到现在：[一句扎心的原文引用]。"
```

**要素检查：** 时间 ✓、身份 ✓、具体数字 ✓、行动 ✓、意外结果 ✓、引用台词 ✓

### 模式B：客户/朋友的故事（每周不超过1篇）

与模式A相同结构，但人物必须是"我认识的某个人"而非"我自己"。必须给人物一个区别于"做了X年的XX"的标识——具体性格特征、窘境画面、或只有当事人才知道的细节。

### 模式C：真实数据冲突

以一个反直觉的数据开头，制造认知冲击：
- "我统计过43个高管招聘案例，发现了一个让我后背发凉的数字——"
- "我复盘了17个重大决策案例，发现'结果好'和'结果差'的两组之间只有一个区别——"

### 模式D：公开案例拆解（每周至少1篇）

以公开已知的企业事件切入，但带入自己的分析视角。

## 四、标题公式（每日轮换，不连续两天用同一种）

| 编号 | 句式 | 示例 |
|------|------|------|
| 1 | 你[做了某件事]，结果[意外结果]——真正的原因是[反常识归因] | 你每个月看报表觉得还行，年底利润少了40%——不是市场不好，是你看到的每个数字都是过去式 |
| 2 | [一个多数人正在做的事]，正在让[一个重要结果]越来越远 | 你越盯着新高管的业绩数字看，他越接不住 |
| 3 | 你以为[表面现象]是因为[常见归因]，真正的原因是[意外归因] | 你以为换高管最危险的是人走了业务塌了，真正的原因是你从来没做过换人前的三个月准备 |
| 4 | [矛盾型事实]——这意味着[反常识推论] | 面试评分和入职表现的相关性只有0.21——你凭感觉选人的准确率跟抛硬币差不多 |
| 5 | 你做了[具体行动]，但[关键前提]根本没人检验过 | 你每次做重大决定都只有A方案，等出了事才发现B和C根本没人想过 |

**标题定稿前必做矛盾压测：** 读者看到这个标题的第一反应有没有"不可能吧"？如果没有，重写。

## 五、金句规则（每篇3-5句，强制）

### 金句特征
- 短（15-30字）
- 有转折（"你以为X，其实Y"）
- 说出读者想说但说不出的话
- 适合截图转发

### 「说不出口的话」规则（每篇至少1句，必须放标题或前3段）

找到那句读者心里一直想说但不敢说的话。方法：每写完一段方法论，反问自己——"这个方法论对应的那个不敢面对的真相是什么？"

**好的"说不出口的话"示例：**
- "你不是真的觉得这个候选人没问题。你是太想赶紧把这个人定下来了——招高管太累了。你的疲惫在做判断，不是你的理性。"
- "你不是不会看数据。你是在用'分开看每一个指标'的方式，安慰自己一切正常。"
- "你不是不想听反对意见。你是不知道你的团队里，谁是真同意、谁是没想明白所以同意、谁是想反对但不敢。"

### 禁止的金句类型
- 已经被说烂的句子（"不是AI取代XX，是会用AI的XX取代不会用的XX"）
- 鸡汤型（"努力的人运气不会太差"）
- 恐惧型（"你再不XX就要被淘汰了"）

## 六、产品植入规则（≤8%，严格遵守）

### 植入位置（唯一位置）
方法拆解全部结束之后，"划重点"之前。

### 植入格式（3-4句话，不超过150字）

```
以上这套[方法论名称]，就是我设计AI总参谋"[模块名称]"模块的底层框架。我把这些年的[相关经验]做成了系统化的工具：[一句话说工具做什么]。它不是替你做判断——它帮你把判断前的功课做完。

📌 想了解这个工具怎么用，加我微信：anhai66（备注「[模块关键词]」），我把工作台链接发你。
```

### 禁止的植入方式
- 不在文章中间突然提产品
- 不列功能清单（"支持XX、XX、XX"）
- 不说价格
- 不说"购买""下单""优惠"
- 不出现"扫码""点击""立即"

## 七、系列连续性规则

### 篇间衔接
- **开头衔接（1-2句）：** "上一篇我拆解了[上文主题+核心结论]。今天接着讲——"
- **末尾预告（2-3句）：** "下一篇我会拆解[具体主题]——包括[核心看点1]和[核心看点2]"

### 下篇预告必须写明的三要素
1. **具体主题**（不是"继续聊XX"，而是"下次拆解：XX的5个方法"）
2. **核心价值**（"这篇是判断问题，下篇是解决问题"）
3. **核心故事预告**（"我会讲一个XX老板差点亏了XXX万的故事"）

### 禁止
- 模糊预告："敬请期待""下期更精彩"
- 跳题：昨天预告了A，今天写了B

## 八、互动引导规则（每周换式，同一种≤2次/周）

| 类型 | 示例 | 使用限制 |
|------|------|---------|

## 十、每日生成工作流

### 第1步：确认日期和选题
- 获取今天日期（YYYY-MM-DD）
- 读取《系列规划》确认今天轮到哪个模块
- 读取上一天的公众号文章，提取"下篇预告"——今天的主题必须对齐预告

### 第2步：生成公众号文章
- 创建日期文件夹：`公众号文章/{YYYY-MM-DD}/`
- 文件命名：`{YYYY-MM-DD}-{文章主题关键词}.md`
- 严格按本配置的结构、比例、规则执行
- 字数：2500-3000字

### 第3步：生成配套朋友圈（5条）
- 文件命名：`{YYYY-MM-DD}_朋友圈文案_5条.md`
- 保存在同一日期文件夹内
- 人设：Andy，真实、有料、不说教
- 每条80-200字，开头必须是具体场景/动作/数据
- 时间分布：早7:30 / 午11:00 / 午14:00 / 晚17:30 / 晚21:00
- 内容分布：文章预告 / 金句卡片 / 个人踩坑 / 读者互动 / 幕后+下篇预告

### 第4步：质量检查
- [ ] 标题通过"矛盾压测"（读了第一反应有没有"不可能吧"）
- [ ] 开头是具体人物+具体细节+痛点台词（不是观点句）
- [ ] 有150-200字"反思停留"（认知翻转后、方法论前）
- [ ] 3-5句金句，至少1句"说不出口的话"放在标题或前3段
- [ ] 产品植入≤8%，只在方法结束后出现一次
- [ ] 产品描述不超过150字，不带功能清单、价格、购买词
- [ ] 有明确的"下篇预告"（具体主题+核心价值+故事预告）
- [ ] 有互动引导（二选一/开放讲述/反转预判，每周轮换）
- [ ] 禁止词检查通过（无赋能/助力/抓手/闭环/链路/沉淀等）
- [ ] 人名使用"Andy"（不是陈海或其他名字）
