# 总参部 — 产品架构方案（讨论稿）

> 版本：v0.2 | 日期：2026.05.16
> 定位：中小企业老板的"随身参谋部"，基于华为DSTE/BLM战略模型，专注服装/电商行业
> 与AI总参谋的关系：互补产品线。AI总参谋=你的一个参谋（轻量提示词工具）；总参部=你的整个参谋部（方法论驱动的经营管理系统）

---

## -1、产品命名分析

### 命名原则（参考《产品命名与用户习惯深度分析》）

1. **共振型命名**：让目标用户在0.5秒内产生"这就是我要的"，而非描述产品做什么
2. **三层击中**：认知层（对我有什么用）+ 情感层（产生向往/安心）+ 身份层（用这个让我成为什么样的人）
3. **暗示"人"的角色**：用户对人产生信任，对工具产生比价。不是"一个AI工具"，是"一个可以信赖的角色"
4. **去AI化**：2026年"AI"在传播端是负资产——套壳、噱头、不值钱
5. **简短**：3-4字，方便口口相传
6. **升级感知**：名字本身就要让人感觉到"比总参谋更强"

### 命名推演

| 候选名 | 符合"人角色" | 升级感 | 简短 | 问题 |
|--------|------------|--------|------|------|
| AI商业驾驶舱 | ✗ 工具隐喻 | — | ✗ 6字 | AI负资产、"驾驶舱"是东西不是人 |
| 决策总参 | ✓ | 平级 | ✓ 4字 | 和总参谋太接近，"决策"窄化了产品 |
| 经营智囊 | ✓ | 平级 | ✓ 4字 | 和面试智囊同系列但容易混淆层级 |
| 智囊团 | ✓ | ✓ | ✓ 3字 | 暗示团队，但"智囊"偏咨询感 |
| 参谋部 | ✓ | ✓✓ | ✓ 3字 | 军事感偏重，但品牌一致 |
| **总参部** | ✓ | ✓✓✓ | ✓ 3字 | 稍军事化，但品牌已建立 |

### 推荐：**总参部**

**逻辑链**：
```
总参谋 = 我有一个参谋帮我出主意（AI总参谋，产品6）
                   ↓ 升级
总参部 = 我有一整个参谋部帮我管生意（新产品的定位）
```

**三层共振**：
- 认知层：从"一个人帮我"到"一个部门帮我"——经营、战略、计划、人才、风险全覆盖
- 情感层："我不只有一个参谋，我有一个参谋部"——这是指挥官才有的配置
- 身份层：用总参部的老板 = 有全套指挥体系的决策者，不是光杆司令

**与现有产品的品牌矩阵**：

| 产品 | 名字 | 角色隐喻 | 目标用户 | 一句话 |
|------|------|---------|---------|--------|
| 产品5 | 面试智囊 | 你的招聘军师 | HR | 面试判断有人商量 |
| 产品6 | AI总参谋 | 你的随身参谋 | 老板 | 遇到经营问题，找个参谋问问 |
| 产品7（新） | **总参部** | 你的整套参谋部 | 深度老板 | 把整盘生意装进参谋部 |

**对外呈现**：
> 总参部——老板的随身参谋部。经营看板·战略推演·计划解码·人才地图·风险预警，从战略到执行一站式。

**内外部命名区分**：
- 产品名/文件名/页面标题：**总参部**
- 对外沟通/话术/发客户：**总参部**（或"你的参谋部"）
- 激活码后台/管理系统：总参部

> 备选：**参谋部**——含义相同，但"总参部"在与"总参谋"的升级关系上更直接。如果觉得"总参部"太军事化，可降为"参谋部"。

## 〇、与现有AI总参谋的差异化定位

| 维度 | AI总参谋（现有） | 总参部（新） |
|------|-----------------|-------------------|
| **核心逻辑** | 填表单 → 生成提示词 → 复制到AI | 方法论驱动 → 结构化数据输入 → 参谋部看板+AI解读 |
| **战略深度** | 无战略框架，散点式分析 | 华为BLM/DSTE模型打底，完整战略→执行链路 |
| **行业适配** | 通用型（任何行业） | 服装/电商行业深度定制（6种战略模式） |
| **使用方式** | 单次使用，每次重新填 | 持续使用，数据沉淀，趋势追踪 |
| **输出形态** | AI提示词文本 | 仪表盘看板 + AI分析报告 + 预警清单 + 行动建议 |
| **人才模块** | 人才盘点 + 面试题库/分析 + 试用辅佐 | 人才盘点 + 离职预警 + 人效分析 + 高管招聘全流程（三合一） |
| **计划模块** | 无 | 华为BEM战略解码：战略地图 → KPI → 重点工作 → PBC |
| **风险模块** | 无独立模块 | 独立风险中心：风险识别 → 评估矩阵 → 应对方案 |
| **比喻** | AI总参谋 = 你有问题，找个参谋问问 | 总参部 = 你坐进指挥室，参谋班子全到位 |
| **客户重叠** | 适合首次接触AI的老板 | 适合已经用过AI总参谋、想系统化的老板 |

**一句话差异**：AI总参谋帮你"问AI一个问题"，总参部帮你"管好整盘生意"。

## 二、经营中心 — "看清现在"

### 设计理念
> 老板每天3分钟看清经营全貌，不靠感觉靠数据。

### 核心功能

#### 2.1 经营仪表盘（首页）
```
┌─────────────────── 今日经营概览 ───────────────────┐
│                                                     │
│   ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌───────┐ │
│   │ 营收     │  │ 毛利     │  │ 现金流   │  │ 人效   │ │
│   │ 82.5万   │  │ 38.2%   │  │ +12.3万  │  │ 3.2万  │ │
│   │ ↑5%     │  │ ↓2%     │  │ ↑8%     │  │ →平    │ │
│   └─────────┘  └─────────┘  └─────────┘  └───────┘ │
│                                                     │
│   健康度评分：72分 ⚠  │  较上月 ↓3分               │
│   关注项：毛利连续2月下滑 │ 库存周转放缓 │ 人效持平  │
└─────────────────────────────────────────────────────┘
```

#### 2.2 经营分析（分层递进，复用AI总参谋框架）
- **第一层**（核心3字段）：营收、成本、当前最头疼的问题
- **第二层**：财务深度（毛利率、费用率、现金流、ROI）
- **第三层**：业务增长（新客、复购、客单价、转化率）
- **第四层**：运营效率（库存周转、人效、坪效）
- **第五层**：人力效能（人均产出、薪酬占比、关键岗位空缺率）

#### 2.3 经营早报（每日3分钟）
- 复用AI总参谋的早报框架，但增加：
  - 数据趋势自动对比（环比、同比）
  - 昨日数据+今日关注+竞品动态 → AI生成简报
  - 异常指标自动标红预警

#### 2.4 与AI总参谋的差异
| AI总参谋 | 总参部 |
|---------|------------|
| 填表单 → 生成提示词 → 手动复制到AI | 填数据 → 自动生成看板 + AI解读直接嵌入页面 |
| 每次独立使用，无历史追踪 | 数据存localStorage，趋势图自动生成 |
| 仅提示词输出 | 仪表盘 + 健康度评分 + 趋势图 + 预警 + 提示词 |

## 三、战略中心 — "定好方向"

### 设计理念
> 把华为BLM模型简化适配中小企业，15分钟完成一次有框架的战略思考。

### 底层方法论：华为BLM模型（中小企业简化版）

**华为BLM原版10要素** → **中小企业6要素简化版**：

```
原版BLM                              简化版（总参部用）
─────────────────────────────────────────────────────
【战略侧】
  市场洞察（Market Insights）    →  ① 市场扫描（行业趋势+竞品+客户）
  战略意图（Strategic Intent）   →  ② 战略目标（3年愿景+今年目标）
  创新焦点（Innovation Focus）   →  ③ 增长引擎（选1-2个增长路径）
  业务设计（Business Design）    →  ④ 业务设计（客户/价值/盈利/壁垒）

【差距分析】                      →  ⑤ 差距分析（业绩差+机会差）
  业绩差距 / 机会差距

【执行侧】
  关键任务（Key Tasks）          →  （进入计划中心）
  人才（Talent）                 →  （进入人才中心）
  组织（Organization）
  文化氛围（Culture/Climate）
```

### 核心功能

#### 3.1 行业速配（服装/电商专属）
- 进入战略中心，先选行业类型：服装零售 / 服装电商 / 服装品牌 / 服装供应链
- 再从6种战略模式中勾选匹配的（复用产品4速查卡）：
  - 门店扩张型 / 电商突破型 / 品牌升级型 / 全渠道融合型 / 供应链驱动型 / 组织升级型
- 系统自动预填推荐的增长路径和关键岗位清单

#### 3.2 BLM战略五步引导

**第1步：市场扫描**（5分钟）
- 你的市场在变大还是变小？
- 谁是你的主要竞争对手？他们的优劣势？
- 你的客户有什么变化？（年龄/消费力/购买渠道）

**第2步：战略目标**（3分钟）
- 未来1-3年的营收目标
- 最重要的1-2个突破方向
- 一句话说清你的战略

**第3步：增长引擎**（3分钟）
- 从服装/电商6种增长路径中选择
- 每选一个路径，系统追问3个"关键成功条件"
- 举例：选"电商突破型" → 追问：平台运营/内容流量/线上商品/数据分析，哪项最强最弱？

**第4步：业务设计**（3分钟）
- 客户选择：你的核心客户是谁？不做什么客户？
- 价值主张：客户为什么选你不选竞品？
- 盈利模式：钱从哪赚？主要成本在哪？
- 战略控制点：别人短期抄不走的是什么？

**第5步：差距分析**（2分钟）
- 业绩差距：今年目标和现状之间差多少？
- 机会差距：看到了什么机会但抓不住？

#### 3.3 年度战略模板

**输出：战略一页纸**
```
┌─────────────────────────────────────────────┐
│            XX公司 2026年度战略一页纸          │
├─────────────────────────────────────────────┤
│ 战略目标：从1.5亿→2.5亿，线上占比从20%→40%    │
│ 增长引擎：电商突破（抖音+私域）+ 品牌升级      │
│ 核心客户：25-35岁都市女性，客单价300-800元     │
│ 价值主张：通勤+休闲全场景，性价比最高的原创设计 │
│ 战略控制：50家工厂深度绑定，7天快反供应链       │
│ 关键差距：缺电商操盘手、缺品牌总监、库存不准    │
└─────────────────────────────────────────────┘
```

#### 3.4 与AI总参谋的差异
| AI总参谋 | 总参部 |
|---------|------------|
| 无独立战略模块 | 完整BLM五步法战略制定 |
| 通用于任何行业 | 服装/电商6种战略模式预置+行业话术 |
| 无战略模板 | 年度战略一页纸模板 + 情景推演对比 |
| 决策沙盘只做方案推演 | 战略中心做"方向选择"+"增长路径设计"+"推演" |

## 四、计划中心 — "拆到动作"

### 设计理念
> 用华为BEM战略解码六步法，把战略变成每个人的具体动作。

### 底层方法论：华为BEM（业务执行力模型）简化版

**华为BEM原版6步** → **中小企业3步简化版**：

```
原版BEM                                简化版（总参部用）
─────────────────────────────────────────────────────
第1步：战略澄清                   →  ① 战略到年度目标（填3-5个核心目标）
第2步：绘制战略地图               →      ↑ 合并为一步
第3步：导出KPI                    →  ② 目标到KPI（每个目标2-3个KPI）
第4步：CTQ-Y关键质量特性分析      →      ↑ 合并为KPI拆解
第5步：导出重点工作               →  ③ KPI到动作（谁/做什么/何时完成/PBC）
第6步：制定PBC（个人绩效承诺）     →      ↑ 合并，含责任人+截止日期
```

### 核心功能

#### 4.1 从战略自动导入
- 定好战略一页纸后 → 一键导入计划中心
- 系统自动把战略目标拆成年度目标模板

#### 4.2 三步拆解

**① 战略 → 年度目标**（从战略一页纸自动导入）
```
战略目标：营收1.5亿→2.5亿，线上占比20%→40%
↓
年度目标1：总营收2.5亿（其中线上1亿）
年度目标2：抖音GMV从0→3000万
年度目标3：私域复购率从15%→30%
年度目标4：品牌认知度进入细分品类前三
```

**② 年度目标 → KPI**
```
目标1：总营收2.5亿 → KPI: 季度营收/月度营收/渠道完成率
目标2：抖音3000万 → KPI: 粉丝数/月均GMV/ROI/爆款率
目标3：私域复购30% → KPI: 企微好友数/社群活跃度/复购率
```

**③ KPI → 关键任务 + 责任人 + PBC**
```
关键任务              责任人    完成时间    PBC承诺
搭建抖音直播间团队     电商总监   6月30日    团队5人到岗，首月GMV≥50万
私域会员体系搭建       CRM经理   7月31日    企微沉淀5万用户，标签体系上线
打造3个品牌爆款       设计总监   9月30日    单款月销破5000件，毛利率≥45%
```

#### 4.3 目标追踪仪表盘
- 每个目标一个进度条（绿灯/黄灯/红灯）
- 月度自动提醒："以下5个目标本月进度滞后，建议关注"
- 季度复盘模板：目标完成情况+偏差分析+下季度调整

#### 4.4 与AI总参谋的差异
| AI总参谋 | 总参部 |
|---------|------------|
| 无计划功能 | 华为BEM三步法：目标→KPI→动作+PBC |
| 无目标追踪 | 进度条+月度预警+季度复盘 |
| 无责任人机制 | 每个关键任务指定责任人+截止日期 |

## 五、人才中心 — "搞定谁干"

### 设计理念
> 把高管招聘全流程合并，新增离职预警和人效分析，打造完整的人才管理闭环。

### 模块结构

```
人才中心
├── 5.1 人才盘点（九宫格，复用产品4方法论）
├── 5.2 离职预警（新增）★
├── 5.3 人效分析（新增）★
├── 5.4 高管招聘全流程（三合一：出题+面试分析+试用辅导）
└── 5.5 人才数据看板
```

#### 5.1 人才盘点
- 九宫格评估（业绩×潜力）
- 关键岗位清单 + 现有人员评估
- 能力缺口分析
- **复用产品4人才盘点方法论，升级为在线工具**

#### 5.2 离职预警（新增）★
```
评估维度（每维度1-5分）：
├── 工作状态：最近是否表现异常？（迟到早退/参与度下降/推掉重要任务）
├── 外部诱惑：猎头接触频率？行业在挖什么人？
├── 内部发展：在公司还能学到什么？有上升空间吗？
├── 薪酬竞争力：和市场比处于什么水平？
├── 关系状态：和老板/同事的关系是否出现裂痕？
└── 个人变化：家庭/生活/城市是否有变动计划？

输出：
├── 离职风险TOP3名单（带风险等级红/黄/绿）
├── 每人的离职触发因素分析
├── 挽留策略建议（加薪/调岗/授权/股权/谈心）
└── 如果真走了，继任方案（内部提拔谁？还是外招？）
```

#### 5.3 人效分析（新增）★
```
分析维度：
├── 人均产出：人均营收/人均利润（对标同行业）
├── 薪酬效率：每1元薪酬产生的营收（薪酬费效比）
├── 结构健康度：前后台人员比例、管理层级深度
├── 增速匹配：人员增速 vs 营收增速（是否人浮于事）
├── 关键岗位空缺率：战略相关的核心岗位有多少空缺
└── 流失成本：过去12个月关键人才流失造成的影响

输出：
├── 人效体检报告（6维雷达图）
├── 与行业均值对比
├── 改进建议（裁/招/调/培）
```

#### 5.4 高管招聘全流程（三合一）

**整合AI总参谋的三个独立模块为一个完整流程**：

```
高管招聘全流程
    │
    ├── 阶段1：面试出题
    │   ├── 5大维度24题精准出题（复用AI总参谋题库）
    │   ├── 根据岗位画像自动推荐题目组合
    │   └── 每道题含：考察点+好回答标准+递进追问
    │
    ├── 阶段2：面试分析
    │   ├── 四维打分（业绩/战略/领导力/文化）
    │   ├── 面试记录分析+证据链
    │   ├── 红旗信号识别
    │   ├── 背调方案生成
    │   └── 候选人对比分析（雷达图）
    │
    └── 阶段3：试用辅佐
        ├── 90天融入计划（比AI总参谋更详尽）
        ├── 第30/60/90天检查点+老板行动清单
        ├── 试用期风险早期预警
        └── 止损决策框架（第60天决定留/看/走）
```

**整合后的优势**：
- 一个入口，一个流程，不用跳三个模块
- 数据贯通：面试时对该候选人的评估 → 自动对接试用期的关注重点
- 举例：面试时发现该高管"战略思维强但团队管理弱" → 试用期自动生成"团队融入专项计划"

#### 5.5 人才数据看板
- 现有人才九宫格可视化
- 离职风险热力图
- 人效趋势追踪（月度）
- 高管招聘管道状态（寻访中/面试中/发Offer中/试用中）

#### 5.6 与AI总参谋的差异
| AI总参谋 | 总参部 |
|---------|------------|
| 人才盘点仅有九宫格 | 九宫格+能力缺口+关键岗位清单 |
| 无离职预警 | 6维离职风险评估+挽留策略+继任方案 |
| 无人效分析 | 6维人效体检+对标+改进建议 |
| 面试题库/分析/试用分散为3个模块 | 合并为"高管招聘全流程"一个入口 |
| 数据不通 | 面试评估→试用计划自动对接 |

## 六、风险中心 — "兜住底线"

### 设计理念
> 不只看机会，也看风险。把"出了问题怎么办"变成"提前知道可能出什么问题"。

### 核心功能

#### 6.1 风险识别
```
风险分类（服装/电商行业定制）：
├── 市场风险：需求下滑、竞品降价、平台政策变化
├── 供应链风险：面料断供、工厂延期、质量事故
├── 人才风险：核心高管离职、关键岗位空缺（对接人才中心）
├── 财务风险：现金流断裂、库存积压贬值
├── 战略风险：新业务失败、转型踩空
└── 合规风险：税务、劳动、知识产权
```

#### 6.2 风险评估矩阵
- 每个风险按「发生概率 × 影响程度」打分
- 自动生成风险热力图（红/黄/绿）
- 重点关注"高概率+高影响"象限

#### 6.3 情景推演（从AI总参谋决策沙盘升级）
- 不只推演"方案A vs 方案B"，也推演"如果XX风险发生"
- 示例："如果最大的代工厂突然断供，你有什么备选方案？"
- 前提条件检验：你的战略依赖哪些前提假设？如果假设错了怎么办？
- 输出：风险应对预案 + 触发机制（达到什么指标就启动预案）

#### 6.4 与AI总参谋的差异
| AI总参谋 | 总参部 |
|---------|------------|
| 决策沙盘仅做方案推演 | 风险中心做"全盘风险识别+评估+应对" |
| 无行业定制风险清单 | 服装/电商6大类风险预置清单 |
| 无前提条件检验 | 每个战略目标可做"前提条件压力测试" |
| 独立模块 | 与战略/计划/人才中心联动（战略定方向→风险中心标雷区） |

## 七、跨中心联动设计

### 核心设计原则：五大中心不是孤岛，是联动的

```
经营中心 ──→ 发现经营异常 ──→ 触发战略复盘（战略中心）
     │
     经营数据 ──→ 输入到人效分析（人才中心）
     
战略中心 ──→ 定好方向 ──→ 自动导入计划中心
     │
     战略需要的能力 ──→ 触发人才缺口分析（人才中心）
     │
     战略假设 ──→ 触发前提条件检验（风险中心）

计划中心 ──→ 关键任务 ──→ 对应到具体岗位（人才中心）
     │
     高风险目标 ──→ 触发风险预警（风险中心）

人才中心 ──→ 离职预警 ──→ 触发"如果核心高管离职"情景推演（风险中心）
      │
      人效下降 ──→ 触发经营分析预警（经营中心）

风险中心 ──→ 识别出的高概率风险 ──→ 进入计划中心的应对任务清单
```

### 联动示例
> 场景：老板发现毛利连降2个月
>
> 经营中心 → 毛利下降 → 自动提示"可能是三个原因：①定价问题（战略中心）②成本问题（经营中心深度分析）③人效下降（人才中心）"
> → 老板点击"去战略中心做复盘"→ 战略中心调出当前战略一页纸 → 检查"低价竞争策略是否还适用"
> → 如果决定提价做品牌 → 计划中心拆解为年度目标 → 人才中心判断需要品牌总监

## 八、与AI总参谋的客户路径设计

```
客户路径设想：

阶段1：首次接触安迪AI调度师

  AI总参谋 → 获客入口、轻量工具、培养用户习惯
  总参部 → 利润产品、系统方案、深度绑定
```

## 九、技术实现方案

### 9.1 技术选型
- **单HTML文件**：与现有产品一致，直接部署GitHub Pages
- **数据存储**：localStorage + IndexedDB（数据量比AI总参谋大，需要结构化存储）
- **图表**：Chart.js CDN（仪表盘、趋势图、雷达图、热力图）
- **AI能力**：仍采用"生成提示词 → 用户复制到DeepSeek/Kimi"模式，但在关键节点提供一键复制优化版提示词

### 9.2 UI设计
- 左侧导航栏：五大中心 + 首页看板
- 中间主区域：当前中心的操作界面
- 顶部状态栏：企业名称 + 当前日期 + 快速跳转
- 底部联动提示："经营数据异常 → 建议去战略中心做复盘 [一键跳转]"

### 9.3 免费/付费模式
- **免费模式**：经营中心的经营早报 + 经营分析第一层（3核心字段）+ 人才盘点
- **付费模式**：全部五大中心
- **激活码体系**：复用产品6的激活码管理后台
